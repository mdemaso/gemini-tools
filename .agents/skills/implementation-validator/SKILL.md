# Implementation Validator Skill

This skill performs rigorous verification and ensures that project documentation remains a "Living Document" that accurately reflects the codebase.

## Workflow

1.  **Analyze Context**:
    *   Read `PRD.md`, `TECH_PLAN.md`, and `IMPLEMENTATION_GUIDE.md`.
    *   Examine code changes and run tests.

2.  **Verify Behavioral Correctness**:
    *   Execute automated tests (including those generated by `test-generator`).
    *   Confirm fulfillment of functional requirements.

3.  **Self-Healing Documentation (Consistency Check)**:
    *   After the implementation passes behavioral tests:
        *   **Scan for Mismatches**: Compare the implemented API contracts, data models, and architectural patterns against the `TECH_PLAN.md`.
        *   **Auto-Update**: If the code is correct but the documentation is outdated, perform a "Self-Healing" update:
            *   Rewrite the affected sections in `PRD.md` or `TECH_PLAN.md` to match the truth in the code.
            *   **Audit Log**: Record the change in a `## Change Log` section with the rationale: "Refined during implementation to optimize [X]".

4.  **Identify Follow-up Work**:
    *   Note technical debt, edge cases, or new requirements.
    *   Suggest additions to `WORK_ITEMS.md`.

5.  **Finality**: Mark as passed only when code, tests, and documentation are perfectly synchronized.
